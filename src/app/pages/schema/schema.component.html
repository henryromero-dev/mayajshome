Create Tenant

Purpose: Establishes a new, isolated environment (a "tenant") within MayaJS to store and manage your application's data. Think of it as creating a separate space for your project, complete with its own database and resources.
Why It's Important: Tenants provide separation and organization for different projects or applications, ensuring data security and preventing conflicts between them.

Delete Tenant

Purpose: Removes a tenant and all associated data permanently. This is a destructive action and should be used with caution.
Why It's Important: Deleting a tenant can be useful for cleaning up unused resources or when a project is no longer needed.

Create Schema

Purpose: Defines the structure of your database within a specific tenant. This involves specifying the tables (e.g., "products," "users"), the columns within each table (e.g., "product_name," "price," "email"), and how they relate to each other.
Why It's Important: The schema is the blueprint for your data. It ensures data consistency and integrity, making it easier to store, retrieve, and manage information within your application.

Delete Schema

Purpose: Removes the database schema and all the data contained within it for a specific tenant. This is a destructive action and should be used with caution.
Why It's Important: Deleting a schema might be necessary if you need to start over with a new data structure or if a project is being discontinued.
